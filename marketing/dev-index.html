Developer implementation guide

Flip-a-Thing PDF Viewer

Use Flip-a-Thing to render one or more PDF flipbooks on the same page. Each instance gets its own controls, listeners, loading state, zoom behavior, and PDF source.

GitHub Repo Marketing page

Required Files

Keep the viewer library files together so the relative asset paths resolve correctly.

  • app.js
  • styles.css
  • ../tool/js/minifyed/pdf.min.mjs
  • ../tool/js/minifyed/pdf.worker.min.mjs
  • ../tool/js/page-flip.browser.min.js
  • ../tool/css/page-flip.css
  • images/loading-bar-book.gif, images/icon-open.svg, and images/icon-full-screen.svg

1. Include Dependencies

Add the PageFlip stylesheet, the Flip-a-Thing stylesheet, PageFlip's browser build, and the Flip-a-Thing module script. 

<link rel="stylesheet" href="../tool/css/page-flip.css" />
<link rel="stylesheet" href="../tool/css/styles.css" />

<script src="../tool/js/page-flip.browser.min.js"></script>
<script type="module" src="../tool/js/app.js"></script>

2. Add a Viewer Mount

The auto initializer looks for .pdf-flipbook elements and reads the PDF URL from data-pdf-url. 

<div class="pdf-flipbook" data-pdf-url="../example-pdfs/FGCU_Classic.pdf"></div>

The PDF path is relative to the HTML page where the viewer is mounted, not relative to app.js.

3. Mount Multiple Viewers

You can place multiple viewer mounts on the same page. Each one becomes an independent flipbook instance. 

<div class="pdf-flipbook" data-pdf-url="../example-pdfs/Annual_Report_Concept_Windows_v3_spreads 1.pdf"></div>
<div class="pdf-flipbook" data-pdf-url="../example-pdfs/FFSS.pdf"></div>
<div class="pdf-flipbook" data-pdf-url="../example-pdfs/FGCU_Classic.pdf"></div>

4. Create an Instance Manually

Use the exported factory when you need to control when a viewer is created, such as inside a modal or after a PDF picker changes. 

<div id="viewerMount"></div>

<script type="module">
  import { createPdfFlipbook } from "../tool/js/app.js";

  const mount = document.querySelector("#viewerMount");
  const viewer = createPdfFlipbook(mount, {
    pdfUrl: "../example-pdfs/FGCU_Classic.pdf",
    pdfLinkUrl: "../example-pdfs/FGCU_Classic.pdf"
  });
</script>

Available Options

Pass these values into createPdfFlipbook(mount, options) or set the PDF URL with data-pdf-url for auto initialization.

  • pdfUrl: The PDF file to render.
  • pdfLinkUrl: The URL opened by the accessible view button. Defaults to pdfUrl.
  • title: Optional label used by the generated viewer markup.
  • startZoom: Optional initial zoom value. The viewer resets to 100% after loading.

Deployment Notes

  • Run the page through a local or hosted web server. Browser PDF workers do not behave reliably from file://.
  • Keep the vendor files and images available at the paths used by app.js.
  • Do not reuse IDs in custom markup. The library namespaces generated IDs and listeners per instance.
  • Use styles.css to customize the viewer controls, toolbar layout, loading image, and responsive behavior.
